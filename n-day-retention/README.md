## **N-day Retention (D0/D1/D3/D7/D14/D30/D60/D90)** 
В данном случае не самая показательная метрика, так как ограничивается конкретными днями, но это классическая метрика, которую мы далее сравним с Rolling Retention. Она даст представление об удержании пользователей в конкретные дни пользования продуктом. Также в перспективе ее можно будет использовать для проведения A/B тестов.

>Там, где значения равны нулю - это показатели для тех когорт, у которых не прошёл период N дней с момента регистрации.

Глобально в контексте аналитики онлайн платформы Rolling Retention будет более показательным, чем N-Day Retention, ведь если пользователь не пришел именно в те дни, которые принято брать за отсечки при расчете N-Day Retention, но приходил в другие дни, - он, очевидно, является активным, а N-Day Retention его не учтет. Однако на основе результатов N-Day Retention можно сделать **несколько выводов**:
1) На следующий день после регистрации возвращается ~10-15% пользователей - это довольно низкий показатель. Нормальным для платформ с регулярным использованием считается ~20-25%. Возможно, есть проблемы с непонятным интерфейсом, контентом или отсутствием напоминаний.
2) Для апрельской когорты наблюдается скачок 3-Day Retention (13,33) по сравнению с 1-Day Retention (10,48). Это говорит о том, что, скорее всего, в апреле была введена рассылка/пуши, побуждающие пользователей заходить на платформу на 3 день после регистрации.
